
# Local Businesses and Services Finder (Android - Kotlin)

## Overview

As a software engineer, my goal with this project was to strengthen my skills in mobile application development while working with real-world features such as maps, user interaction, and dynamic data filtering. I wanted to build a practical Android application that integrates location-based services and demonstrates how users can explore nearby places visually.

The software I developed is a **Local Businesses and Services Finder Android application** built using Kotlin and Google Maps. The app displays a map centered on the user’s location and includes multiple markers representing different types of local places such as restaurants, hospitals, banks, and cafes.

### How to Use the App

- Launch the application on an Android device or emulator
- View the map centered on the current location (Addis Ababa by default)
- Explore the map with **20+ markers** representing different services
- Tap on any marker to view details about the location
- Use the search bar to filter places by name or category (e.g., "bank", "cafe")

### Data Source

The current implementation uses **locally defined data** within the app. This ensures fast performance and simplicity. The architecture is designed so that it can easily be extended to use live data from APIs in the future.

### Purpose

The purpose of this project was to gain hands-on experience with:

- Google Maps SDK integration
- Handling multiple map markers
- Implementing interactive UI features like search filtering
- Structuring scalable Android applications using Kotlin
